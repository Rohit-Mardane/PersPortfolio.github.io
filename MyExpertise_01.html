Initiative 1: Automated Information Extraction

Description

In this initiative, I proposed and developed an MVP for an automated information extraction application using Python and the RAG module. The solution is designed to automatically generate technical documentation for any software or application. It serves as a generic platform that collates information and transforms it into a structured document draft.


Business Requirements

After observing the role of documentation in the software development lifecycle of companies with high project volumes and large-scale deliverables, it is clear that documentation is often deprioritized, created late in the cycle, or produced inconsistently. This causes delays in releases, incomplete technical references, and extra rework. The issue becomes even more critical in confidential and sensitive projects, where missing or poor documentation directly affects progress and compliance.

Manual documentation is time-consuming, repetitive, and error-prone, even in Agile workflows. Project teams including Project Managers, Business Analysts, Developers, and Quality Analystsseldom maintain documents in a standard format with sufficient technical details. As a result, projects frequently lack the right information at the right time.

To resolve these challenges, the project requires an automated documentation solution that:

  • Works within existing development workflows.
  • Produces standardized, accurate, and up-to-date documents across all stages.
  • Reduces manual effort, rework, and review cycles.
  • Scales across multiple projects and product lines.

Solution Proposed

The MVP was designed to automate documentation by extracting, organizing, and generating structured drafts directly from project resources. Built in Python with a Retrieval-Augmented Generation (RAG) pipeline, the solution combined rule-based preprocessing with AI-assisted draft generation. The goal was to demonstrate how documentation could shift from a manual afterthought to an integrated and intelligent process.


The workflow included the following components:

  1. Content Extraction and Classification Multiple parsing libraries—such as pdfreader, python-docx, and Pandas—handled heterogeneous inputs including PDFs, Word documents, CSVs, and links from repositories like Confluence and SharePoint. Extracted content was normalized and segmented into coherent units, which were then grouped into topics and subtopics through embeddings and clustering techniques.
  2. Node and Super Node Mapping Each extracted unit was represented as a Node, with related Nodes clustered into Super Nodes that reflected higher-level structures such as headings or service groupings. The mapping rules, categories, and schema definitions were stored in SQLite, while JSON/YAML configurations enabled quick adaptation to new projects without code changes.
  3. Feature Tagging
    Metadata—such as document type, information level, and content category—was attached to every Node and persisted in SQLite. This provided structured consistency across all generated drafts and laid the foundation for scalable reuse.
  4. RAG-Based Draft Generation
    The RAG pipeline was prompted with Nodes, metadata, and predefined mapping rules to produce the first draft of the documentation. This integration allowed the generative model to operate within clear structural boundaries, ensuring that the output was both coherent and aligned with the extracted resources.
  5. Confidence Scoring
    Each draft was evaluated against the source materials, producing a composite confidence score broken down into five dimensions:
    • Company Standard Language
    • Document Format
    • Topic Relevance
    • Checklist Compliance
    • Technical Writing Standards
    The scoring relied on a hybrid approach combining regex validation, clustering-based similarity, and rule-based checks, with all criteria defined and managed in SQLite.
  6. Interactive TOC Preview
    A dynamic Table of Contents view allowed users to navigate the draft, open sections, and edit content inline. This streamlined the review cycle and removed the dependency on external editing tools.
  7. Final Output
    The enhanced draft could then be downloaded or shared, ready for further refinement or direct integration into delivery pipelines.

Impact and Scalability

The MVP demonstrated that documentation traditionally consuming a significant share of the development lifecycle can be generated almost instantly in a structured and consistent format. While the first version employed primarily rule-based techniques with RAG augmentation, the architecture is inherently scalable. With deeper NLP integration, adaptive mappings, and machine learning–driven scoring, the framework can evolve into a fully intelligent documentation platform capable of supporting large-scale, enterprise environments.

I received a Wall of Fame Award for the Month of September 2023 as a recognition of my proposal and MVP and added to a cross functional team for RAG Agent development and experimentation.

Initiative 1: RAG-Based Contract Mining

Description

In this initiative, I contributed as a sandbox tester and feature developer who helped improve the user experience and validate the desired results.


Business Requirements

There was a need of information extractor that can parsed through the hugh pile of scan document stored in different data sets and provide the information based on the prompt shared by the user. These information includes contract numbers, dates, parties involved, and key terms, etc. Since most of the documents are unstructured and stored in various formats, extracting relevant information poses a significant challenge. Some documents may contain handwritten notes, while others are scanned images with varying degrees of clarity.

To resolve these challenges, the initiative requires a data mining solution that:

  • Enables user to select repositories or data sources for analysis.
  • Enables to write a query or prompt to extract specific information.
  • Enables user to export results in various formats.
  • Should not store the data once the session ends.

Solution Proposed

The team built a stateless, session-scoped RAG pipeline that ingests mixed-format contract files (PDF scans, images, DOCX, CSV), performs OCR + layout parsing, normalizes content, and answers user prompts by retrieving relevant passages and extracting structured fields (contract number, dates, parties, key terms). The system deletes all artifacts at session end.


The workflow included the following components:

  1. Data Ingestion Users could upload documents or select repositories such as SharePoint, Confluence, or local datasets. Each session was isolated and temporary, ensuring that no data was stored beyond its lifecycle.
  2. Preprocessing and OCR The system handled different formats—PDFs, Word documents, CSVs, and scanned images. OCR engines processed low-quality scans and handwritten notes, while layout parsing preserved tables and headers. All extracted text was normalized into consistent chunks with metadata such as file name and page reference.
  3. Content Indexing
    Text chunks were embedded using transformer-based models and stored in a session-only in-memory vector index. This enabled semantic retrieval without persisting any information after the session.
  4. Query and Retrieval
    When a user entered a prompt (for example, “List the contract numbers and effective dates”), the system retrieved the most relevant text segments. A hybrid approach combined semantic similarity with lightweight rule-based extraction to ensure accuracy.
  5. Field Extraction
    Critical fields such as contract numbers, dates, and parties were identified using a mix of regex patterns, named-entity recognition, and proximity-based matching. Key terms and clauses were detected using keyword and embedding similarity, ensuring coverage of standard contract elements like termination, confidentiality, and governing law.
  6. Results and Preview
    The output was presented in a structured table with values, confidence scores, and citations linking back to the original page or section. An interactive preview allowed users to review results before exporting.
  7. Export and Cleanup
    Results could be exported in multiple formats, including CSV, Excel, or JSON. At the end of the session, all temporary data—files, indexes, and caches—was automatically deleted to maintain security and compliance.

Impact and Scalability

The MVP demonstrated how unstructured contract data, often scattered and inconsistent in quality, can be transformed into reliable, queryable information. By combining OCR, rule-based extraction, and a RAG pipeline, the solution offered both flexibility and accuracy while remaining stateless. The design also lays a foundation for scaling into a more intelligent contract analysis platform by introducing layout-aware models, domain-specific ontologies, and learning-based confidence scoring in future iterations.
